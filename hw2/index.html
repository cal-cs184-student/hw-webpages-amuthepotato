Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm is a recursive method to evaluate Bezier curves at a parameter. It repeatedly performs linear interpolation between adjacent control points. With n points, we compute n - 1 new points using the formula \( p^'i = (1 - t)pi + tpi+1 \). These new points form a smaller polygon, and then the points of that polygon become the new points used in the formula. The process repeats until only one point remains. That final point lies on the Bezier curve at parameter t. In our implementation, we wrote a function that performs one step of the interpolation between each pair of neighboring control points and returns the resulting vector between those two points. To get the full curve, we applied this step iteratively until only one point remained, which is returned as the final evaluated point of the curve.

Part 2: Bezier surfaces with separable 1D de Casteljau

For Bezier surfaces, de Casteljau's algorithm uses the seperability of the surface into two parametric directions, u and v. A Bezier surface is defined by an n x n grid of control points, where each row can be seen as defining a Bezier curve in the u direction. To evaluate at a parameter pair, we first apply the 1D version of the de Casteljau algorithm to each row using the parameter u, which reduces each row to a single point. In doing so, we produce a new set of n points along the v direction. These points can then be treated as control points of a second Bezier curve and we can apply de Casteljau again using parameter v. We interpolate until only one point remains again. That final point is the evaluated position on the Bezier surface. In our implementation, we reused the 1D function to first collapse each row of u, storing the resulting points in a vecotr, and then applied the same 1D algorithm to that vector at v to get the final surface point.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

To implement area-weighted vertex normals, we iterated over all the faces incident to a given vertex using the halfedge data structure. Starting from the vertex's outgoing halfedge, we traversed around the vertex by repeatedly using twin()->next() until returning to the starting halfedge. For each adjacent triangle that was not a boundary face, we computed an area-weighted face normal using the cross product of two edge vectors formed from the triangle’s vertices. This cross product produces a vector whose magnitude is proportional to the triangle’s area and whose direction is perpendicular to the face. We accumulated these vectors for all incident faces and then normalized the final sum to obtain a unit normal at the vertex.

Part 4: Edge flip

To implement the edge flip operation, we first ensured the edge was not on the boundary by checking that both adjacent faces were interior triangles. We then labeled each of the six halfedges involved in the two neighboring triangles, along with their associated vertices, edges, and faces. Since an edge flip doesn't create or delete any elements, we just had to make sure the pointers were reassigned consistently. We changed the two halfedges of the target edge to connect the opposite vertives, updated the next, vertex, and face pointers for all six halfedges and reset the halfedge pointers of the vertices and faces affected to ensure that they referenced valid outgoing halfedges. To debug, we drew out the triangles on an iPad and explicitly labeled all of the edges, vertices, and faces with the variables that we had in our code. This helped prevent logical mistakes. Part 6: Loop subdivision for mesh upsampling

We implemented loop subdivision in upsample by following the recommended approach: compute new vertex positions, split all original edges, flip appropriate edges, and then update final vertex positions.

First, we iterated over all original vertices and computed their new positions using the loop weighting rule. For each vertex, we traversed its neighbors using halfedge iteration, summed their positions, and computed the weighted average using u = 3/16 for degree 3 and u = 3/(8n) otherwise. We stored the result in newPosition and marked all original vertices as isNew = false. Next, we computed the new positions for vertices that would be created by edge splits. For each original edge, we retrieved its two endpoints A and B along with the two opposite vertices C and D, and stored 3/8 * (A + B) + 1/8 * (C + D) in newPosition. We also marked all original edges as isNew = false. To avoid splitting edges multiple times, we first collected all original edges into a separate vector. We then split each of these edges exactly once. After each split, we marked the newly created vertex as isNew = true, assigned it the precomputed newPosition, and marked the new edges created during the split as isNew = true. After all splits, we flipped any edge marked as new that connected one old vertex and one new vertex. This step enforces the proper loop subdivision connectivity pattern. Finally, we copied all stored newPosition values into position.

One debugging challenge was only flipping edges connected to the new vertex that connected an old vertex and an old vertex. We did this by skipping every other edge from the new vertex.

After several iterations of loop subdivision, meshes become significantly smoother. Sharp edges and corners round off, and blocky shapes soften. This can be seen below with the teapot.
